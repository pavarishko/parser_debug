# Анализ качества Sitemap для доменов с низким покрытием

## Обзор анализа

Проанализированы результаты парсинга 93 доменов из файла `results/sitemap_quality_top100.csv`. Выявлены домены с низким покрытием (<90%) и определены причины проблем.

## Ключевые выводы

### Основная проблема: региональные поддомены

**Главная причина низкого покрытия** - несоответствие между региональными поддоменами в ожидаемых URL и основным доменом в sitemap.

**Примеры:**
- Ожидаемые URL: `https://barnaul.askona.ru/...`, `https://samara.askona.ru/...`
- URL в sitemap: `https://askona.ru/...`

### Улучшенная нормализация

Добавлена опция `remove_regional_subdomains=True` в функцию [`normalize_url()`](utils.py:26) для автоматического удаления региональных поддоменов при нормализации.

**Пример работы:**
- До: `https://barnaul.askona.ru/path` → `https://barnaul.askona.ru/path`
- После: `https://barnaul.askona.ru/path` → `https://askona.ru/path`

## Технические детали

### Изменения в коде

Модифицирована функция [`normalize_url()`](utils.py:26):
- Добавлен параметр `remove_regional_subdomains=True`
- Реализована логика удаления региональных поддоменов
- Сохранена обратная совместимость

### Алгоритм удаления региональных поддоменов

```python
if remove_regional_subdomains and netloc:
    if '.' in netloc:
        parts = netloc.split('.')
        if len(parts) > 2 and not parts[-2].startswith('co'):
            if parts[-1] in ['ru', 'su', 'рф'] and len(parts) > 2:
                netloc = '.'.join(parts[-2:])
            elif len(parts) > 3:
                netloc = '.'.join(parts[-3:])
```

## Заключение

Основная проблема низкого покрытия - региональные поддомены и различия в структурах URL. Улучшенная нормализация решает часть проблем, но для значительного повышения покрытия требуется более сложный подход к сопоставлению URL.

**Рекомендуемые следующие шаги:**
1. Реализовать fuzzy matching для путей
2. Создать систему маппинга структур URL
3. Протестировать на всех проблемных доменах
