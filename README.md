# All Directory

This directory contains all project-related code, scripts, and results.

## Directory Structure

- **`parsers/`** - Parser modules for sitemap and robots.txt parsing
- **`results/`** - Analysis results, reports, and CSV files
- **`scripts/`** - Utility scripts for data processing and analysis

## About the `data/` Directory

The `data/` directory is located at the root level (outside this `all/` folder) because it contains large cached data files that should not be stored in the git repository.

### Data Directory Contents

The `data/` directory contains files in the format `domain.jsonl`, where each file stores all parsed URLs from all discovered sitemaps for that domain.

**Example:** `data/sitemaps_top100_cache/4lapy.ru_urls.jsonl`

```jsonl
"https://4lapy.ru/product/-38838-1011445"
"https://4lapy.ru/product/-39203-1011853"
"https://4lapy.ru/product/-39582-1012380"
```

### Why `data/` is Separate

- **Size**: The cached sitemap data can be very large (hundreds of megabytes or more)
- **Git Repository**: Large binary or data files should not be committed to version control
- **Regeneratable**: The data can be regenerated by running the sitemap parsing scripts
- **Storage**: Keeping it separate makes it easier to exclude from git using `.gitignore`

### Data Subdirectories

- `data/sitemaps_top100_cache/` - Cached URLs from top 100 domains
- `data/sitemaps_top100_cache_2026-03-18/` - Historical cache from March 18, 2026

## Usage

To regenerate the data in the `data/` directory, run the appropriate scripts from the `scripts/` directory. The parsers in `parsers/` will fetch and parse sitemaps, storing the results in the `data/` directory.
